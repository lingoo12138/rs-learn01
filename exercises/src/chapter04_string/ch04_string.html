第04章：切片与字符串

学习目标

完成本章后，你将能够：

  • 区分 String（堆分配的可变字符串）和 &str（字符串切片）
  • 使用字符串切片安全地引用字符串的一部分
  • 使用 + 运算符和 format! 宏拼接字符串
  • 使用 len()、chars()、contains()、replace() 等常用方法
  • 理解 UTF-8 编码和 Rust 中字符串的底层存储方式
小提示：如果你来自其他语言，注意 Rust 的字符串处理与其他语言有显著差异。在 C++ 中，std::string 可修改且以空字符结尾；在 Java/Python 中，字符串是不可变的，且底层是 UTF-16 或字节序列。Rust 的 String 是可变的、UTF-8 编码的堆分配字符串，而 &str 是不可变的字符串切片视图。理解这两者的区别是 Rust 学习的关键一步。

1. String 与 &str

1.1 String — 堆分配的字符串

String 是一个可增长、可修改、拥有所有权的 UTF-8 字符串类型。它在堆上分配内存，因此长度可以在运行时动态变化。

let mut s = String::from("hello"); s.push_str(" world"); println!("{}", s); // 输出: hello world

1.2 &str — 字符串切片

&str 是一个对字符串数据的不可变引用（切片）。它不拥有数据，只提供对现有字符串数据的视图。字符串字面量就是 &str 类型。

let greeting: &str = "hello"; // 字符串字面量是 &str let s = String::from("hello"); let slice: &str = &s[..]; // 从 String 创建切片
注意：String 和 &str 之间的关系可以理解为：String 是数据的所有者，&str 是数据的借阅者。&str 只保存一个指针和长度信息，不拥有底层数据。

2. 字符串切片（String Slices）

字符串切片是对字符串中一部分内容的引用。切片的范围使用 [起始索引..结束索引] 语法表示，结束索引不包含在切片内。

let s = String::from("hello world"); let hello = &s[0..5]; // "hello" let world = &s[6..11]; // "world" let whole = &s[..]; // "hello world" — 完整切片 let from_start = &s[..5]; // "hello" — 从开头到索引 5 let to_end = &s[6..]; // "world" — 从索引 6 到结尾

切片索引必须落在有效的 UTF-8 字符边界上，否则程序会 panic：

let s = "你好世界"; // &s[0..1] — 错误！"你"占用 3 个字节，索引 0..1 不是字符边界 let hello = &s[0..3]; // "你" — 正确，3 字节

3. 字符串拼接

3.1 使用 + 运算符

+ 运算符调用 add(self, s: &str) -> String 方法，注意左侧必须是 String 类型（所有权被移动）。

let s1 = String::from("Hello"); let s2 = String::from(" World"); let s3 = s1 + &s2; // s1 的所有权被移动到 s3 // println!("{}", s1); // 错误！s1 已失效 println!("{}", s3); // 输出: Hello World

3.2 使用 format! 宏

format! 宏更灵活，不会移动任何变量的所有权：

let s1 = String::from("Hello"); let s2 = String::from("World"); let s = format!("{} {}!", s1, s2); println!("{}", s); // 输出: Hello World! // s1 和 s2 仍然有效！
推荐：在需要拼接多个字符串时，format! 是更好的选择。它的语法和 println! 相同，但不会把内容打印到屏幕，而是返回一个 String。使用 + 运算符时需要注意所有权的移动，使用多个 + 串联时会变得难以阅读。

4. 常用字符串方法

方法 说明 示例
len() 返回字符串的字节数（不是字符数） "hello".len() = 5
chars() 返回字符迭代器 "你好".chars() = 2 个 char
contains() 检查是否包含子串 "hello".contains("ll") = true
replace() 替换所有匹配的子串 "hello".replace("ll", "r") = "hero"
push_str() 追加字符串切片 s.push_str(" world")
is_empty() 检查字符串是否为空 "".is_empty() = true
let s = "hello world"; println!("len: {}", s.len()); // 11 println!("contains 'world': {}", s.contains("world")); // true println!("replace: {}", s.replace("hello", "hi")); // "hi world" for c in s.chars() { print!("{} ", c); // h e l l o w o r l d }

5. UTF-8 与字符串内部表示

Rust 的 String 和 &str 内部都是 UTF-8 编码的字节序列。这意味着：

  • len() 返回的是字节数，不是字符数
  • 一个"字符"（Unicode 标量值）可能占用 1 到 4 个字节
  • 不能使用 s[i] 索引访问字符（Rust 的字符串不是字符数组）
  • 遍历字符需要使用 .chars() 方法
let s = "你好世界"; println!("字节长度: {}", s.len()); // 12（每个汉字 3 字节） println!("字符个数: {}", s.chars().count()); // 4 // 不能这样做！ // let c = s[0]; // 编译错误！ // 正确的做法： for c in s.chars() { println!("{}", c); // 你 好 世 界 }
重要：Rust 选择 UTF-8 作为字符串的内部编码，而不是像 Java 那样使用 UTF-16 或像 Python 3 内部使用灵活编码。这意味着处理 ASCII 文本时非常高效（每个字符 1 字节），但也意味着按索引随机访问字符是 O(n) 操作，而非 O(1)。这是 Rust 在设计上做出的权衡——内存效率优先于随机访问性能。对于需要随机访问字符的场景，Rust 提供了 .chars() 迭代器。

  7. 字符串替换 — 将字符串中的 "hello" 替换为 "hi"。
  8. 字符计数 — 返回字符串中的字符（Unicode 标量值）个数。
