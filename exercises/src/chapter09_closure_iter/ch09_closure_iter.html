第09章：闭包与迭代器

学习目标

完成本章后，你将能够：

  • 理解闭包的语法和类型推导
  • 理解闭包如何捕获环境变量
  • 区分 Fn、FnMut、FnOnce 三种闭包 trait
  • 使用 Iterator trait 的 next 方法
  • 使用 map、filter、fold、collect 等迭代器适配器
  • 将迭代器组合成高效的数据处理链
小提示：闭包和迭代器是 Rust 函数式编程的两大基石。如果你有 JavaScript 背景，闭包就像箭头函数（() => {}），map/filter 则与 JS 数组方法如出一辙。Rust 的迭代器是惰性的——它们直到你调用 collect 或其他消费者方法时才真正执行，这使 Rust 的迭代器链非常高效（编译器通常能优化到与手写循环一样快）。

1. 闭包（Closures）

1.1 闭包基础语法

闭包是可以捕获环境的匿名函数。语法比普通函数更简洁：

// 普通函数 fn add_one(x: i32) -> i32 { x + 1 } // 等价的闭包 let add_one = |x: i32| -> i32 { x + 1 }; // 类型推导（最常见的形式） let add_one = |x| x + 1; println!("{}", add_one(5)); // 6

闭包的语法特点：

  • 参数放在 || 中（像没有括号的箭头函数）
  • 函数体可以用 {} 括起来（多语句时必需），也可以省略大号（单表达式）
  • 参数和返回值的类型注解可以省略——编译器会从使用上下文推导

1.2 捕获环境变量

闭包可以"捕获"定义它的作用域中的变量——这是它与普通函数最本质的区别。

let x = 10; let add_x = |y| y + x; // 捕获了 x assert_eq!(add_x(5), 15); let prefix = "Hello".to_string(); let greet = |name| format!("{} {}", prefix, name); // assert_eq!(prefix, "Hello"); // 编译错误！prefix 已被闭包借用
注意：闭包捕获变量有三种方式——不可变借用（&T）、可变借用（&mut T）、获取所有权（T）。闭包默认使用最"温和"的捕获方式。如果需要获取所有权，使用 move 关键字：move |y| y + x。这在多线程编程中尤其重要。

2. Fn / FnMut / FnOnce

每个闭包都实现了以下 trait 之一（或多个），它们决定了闭包如何捕获变量：

Trait捕获方式调用次数
FnOnce获取所有权（consume）只能调用一次
FnMut可变借用可多次调用（可变）
Fn不可变借用可多次调用（不可变）

继承关系：Fn : FnMut : FnOnce（实现 Fn 的一定实现了 FnMut 和 FnOnce，反之不一定）

// 接受 Fn 闭包（最通用的） fn call_fn<F: Fn(i32) -> i32>(f: F, x: i32) -> i32 { f(x) } let square = |x| x * x; assert_eq!(call_fn(square, 5), 25); // Fn: 不捕获或不可变借用

3. 迭代器（Iterator）

3.1 Iterator Trait

迭代器是 Rust 中处理元素序列的核心抽象。任何实现了 Iterator trait 的类型都是一个迭代器。

trait Iterator { type Item; // 关联类型：迭代器产生的元素类型 fn next(&mut self) -> Option<Self::Item>; }

最基础的方法是 next：它返回下一个元素，如果没有更多元素则返回 None。

let v = vec![1, 2, 3]; let mut iter = v.iter(); // 创建一个迭代器 assert_eq!(iter.next(), Some(&1)); assert_eq!(iter.next(), Some(&2)); assert_eq!(iter.next(), Some(&3)); assert_eq!(iter.next(), None); // 已经没有元素了

3.2 创建迭代器

方法返回说明
.iter()&T 的引用不可变地遍历元素
.iter_mut()&mut T 的可变引用可变地遍历元素
.into_iter()T（所有权）消耗集合，获取元素所有权

4. 迭代器适配器

迭代器适配器（adapter）是消耗一个迭代器并返回另一个迭代器的方法。它们惰性的——直到你调用一个"消费者"方法（如 collect），它们才会执行。

4.1 map：转换每个元素

let nums = vec![1, 2, 3]; let squares: Vec<i32> = nums.into_iter() .map(|n| n * n) .collect(); assert_eq!(squares, vec![1, 4, 9]);

4.2 filter：按条件过滤

let nums = vec![1, 2, 3, 4]; let evens: Vec<i32> = nums.into_iter() .filter(|n| n % 2 == 0) .collect(); assert_eq!(evens, vec![2, 4]);

4.3 fold：归约/累积

fold 接收一个初始累加器和一个闭包，将闭包反复应用到每个元素上，最终返回一个值。

let nums = vec![1, 2, 3]; let sum = nums.into_iter().fold(0, |acc, n| acc + n); assert_eq!(sum, 6); // 0 + 1 + 2 + 3 = 6

4.4 collect：收集为集合

collect 将迭代器转换为集合类型（如 Vec、HashMap、String 等）。

let nums = vec![1, 2, 3]; let doubled: Vec<i32> = nums.iter().map(|n| n * 2).collect(); assert_eq!(doubled, vec![2, 4, 6]);

5. 链式调用与惰性求值

Rust 迭代器的强大之处在于可以链式组合多个适配器，而且因为惰性求值，整个过程通常只需一次遍历。

let nums = vec![1, 2, 3, 4, 5, 6]; // 链式调用：过滤偶数 -> 平方 -> 收集结果 let result: Vec<i32> = nums.into_iter() .filter(|n| n % 2 == 0) // 保留 2, 4, 6 .map(|n| n * n) // 平方: 4, 16, 36 .collect(); assert_eq!(result, vec![4, 16, 36]);
性能理解：上面的代码看起来像创建了多个中间集合（过滤后的数组、平方后的数组），但实际上 Rust 编译器不会分配临时集合！每个适配器只添加一个"变换步骤"，整个链是一个惰性的"配方"。只有在 collect 被调用时，迭代器才实际消耗元素——一次遍历完成所有变换。这被称为"零成本抽象"（zero-cost abstraction）。
