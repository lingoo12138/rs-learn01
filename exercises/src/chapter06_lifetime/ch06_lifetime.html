第06章：生命周期

学习目标

完成本章后，你将能够：

  • 理解为什么需要生命周期标注——防止悬垂引用
  • 使用 'a 语法标注函数中的生命周期参数
  • 在结构体中使用生命周期标注
  • 理解生命周期省略规则（elision rules）
  • 理解 'static 生命周期及其用途
小提示：生命周期（lifetimes）是 Rust 独有的概念，也是新手最感到困惑的部分之一。简单来说，生命周期就是"引用有效的作用域"。当你创建一个引用时，它的生命周期不能超过被引用数据的生命周期。Rust 编译器通过生命周期检查来确保你永远不会创建悬垂引用（dangling reference）。好消息是，大多数情况下编译器可以自动推断生命周期，你只需要在少数复杂情况下显式标注。

1. 为什么需要生命周期？

生命周期的主要目的是防止悬垂引用——即引用指向已经被释放的内存。

let r; // ---------+-- 'a 开始 { // | let x = 5; // -+-- 'b 开始| r = &x; // | | } // -+-- 'b 结束| println!("{}", r); // 编译错误！r 引用了已释放的 x

上面的代码中，r 的生命周期 'a 比 x 的生命周期 'b 长。当代码试图在 'b 结束后使用 r 时，编译器会拒绝——因为 r 变成了悬垂引用。Rust 编译器在编译时通过借用检查器（borrow checker）进行生命周期检查，确保了内存安全。

核心原则：任何引用的生命周期不能超出其引用的数据的生命周期。这是 Rust 保证内存安全的核心机制之一。如果没有生命周期检查，你可能会在不知不觉中访问已经释放的内存，导致程序崩溃或安全漏洞——而这在 C/C++ 中是常见的内存错误。

2. 生命周期标注语法

生命周期参数使用撇号 ' 开头，通常使用小写字母，如 'a、'b。生命周期标注不会改变引用的实际存活时间，只是让借用检查器能够进行正确的分析。

2.1 函数中的生命周期

// 接收两个 &str 引用，返回存活时间较短的那个 fn longest<'a>(x: &'a str, y: &'a str) -> &'a str { if x.len() > y.len() { x } else { y } }

这里 <'a> 声明了一个生命周期参数 'a。函数签名表示：x 和 y 都至少存活了 'a 这么长，返回值也存活了 'a 这么长。实际上返回值的生命周期等于 x 和 y 中较短的那个。

2.2 多个生命周期参数

fn first_and_second<'a, 'b>(first: &'a str, _second: &'b str) -> &'a str { first // 返回值只和 first 的生命周期有关 }

当返回值只与其中一个参数的生命周期相关时，可以为不同的参数标注不同的生命周期。

3. 结构体中的生命周期

当结构体的字段存储引用时，需要为每个引用字段标注生命周期：

struct Excerpt<'a> { part: &'a str, // 引用必须标注生命周期 } fn main() { let novel = String::from("Call me Ishmael..."); let first_sentence = novel.split('.').next().unwrap(); let e = Excerpt { part: first_sentence }; println!("{}", e.part); // 输出: "Call me Ishmael" }

这里 Excerpt<'a> 中的 'a 表示 part 引用的数据至少和 Excerpt 实例存活一样长。这样编译器就能确保 part 不会变成悬垂引用。

// 也可以通过 impl 块为结构体中的方法标注生命周期 impl<'a> Excerpt<'a> { fn get_part(&self) -> &'a str { self.part } }

4. 生命周期省略规则（Elision Rules）

Rust 编译器内置了三条规定，在大多数常见情况下可以自动推断生命周期，无需显式标注：

  1. 每个引用参数都有自己的生命周期参数。
  2. 如果只有一个输入生命周期参数，它会被赋予所有输出生命周期参数。
  3. 如果有多个输入生命周期参数，但其中一个是 &self 或 &mut self，那么 self 的生命周期被赋予所有输出生命周期参数。
// 由于省略规则，以下代码不需要显式标注生命周期 fn first_word(s: &str) -> &str { s.split_whitespace().next().unwrap_or("") } // 编译器会自动将其扩展为： // fn first_word<'a>(s: &'a str) -> &'a str
理解省略规则：省略规则不是"推测"，而是一套确定的算法。如果编译器应用所有规则后仍然无法确定生命周期，就会报错，要求你显式标注。这意味着当看到需要标注生命周期的错误时，往往是因为函数的输入输出关系比较复杂，编译器无法自动确定。

5. 'static 生命周期

'static 是一个特殊的生命周期，表示引用在整个程序运行期间都有效。字符串字面量就是 'static 生命周期：

let s: &'static str = "I live forever";

'static 并不罕见——所有字符串字面量都是 'static。但避免过度使用 'static 作为通用解决方案，它通常意味着你要存储的数据会存活整个程序生命周期。

注意：看到 'static 约束时，不要立即认为引用真的存活了"整个程序"。更准确的理解是：只要程序在运行，被引用的数据就一直有效。将 'static 作为泛型约束用于 trait bound 的情况，与直接标注引用类型为 &'static 的含义有所不同。随着学习的深入，你会逐渐理解这些细微差别。
