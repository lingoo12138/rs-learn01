第10章：智能指针

学习目标

完成本章后，你将能够：

  • 理解 Box<T> 的堆分配和作用
  • 使用 Box 创建递归类型
  • 使用 Rc<T> 实现引用计数共享所有权
  • 使用 RefCell<T> 实现内部可变性
  • 组合 Rc<RefCell<T>> 实现共享可变数据
  • 理解 Rust 智能指针的所有权和借用规则
小提示：智能指针是"行为像指针的数据结构"——它们不仅持有内存地址，还拥有额外的元数据和功能。如果你来自 C++ 背景，Box 类似 std::unique_ptr，Rc 类似 std::shared_ptr。但 Rust 智能指针的区别在于它们在编译时就保证了内存安全和线程安全。

1. Box<T> — 堆分配

1.1 为什么需要 Box？

通常 Rust 变量在栈上分配内存。但有些数据需要在堆上分配：

  • 数据大小在编译时未知
  • 需要超出当前作用域存活的数据
  • 递归类型（需要间接存储）
  • 希望减少大型数据拷贝的开销
// Box 将 i32 放到堆上，返回一个指向堆内存的指针 let b = Box::new(5); println!("{}", b); // 5 (自动解引用) println!("{}", *b); // 5 (显式解引用) // 离开作用域时，Box 会自动释放堆内存

Box<T> 实现了 Deref 和 Drop trait：

  • Deref 允许 Box<T> 像 &T 一样使用（自动解引用）
  • Drop 确保离开作用域时释放堆内存

1.2 Box 和递归类型

Rust 需要在编译时知道每个类型的大小。递归类型如果用值存储，大小是无限的——必须用 Box 间接存储：

enum List { Cons(i32, Box<List>), // Box 是指针，大小固定 Nil, } let list = Cons(1, Box::new(Cons(2, Box::new(Nil))));
为什么 Box 是必要的？没有 Box 时，Cons(i32, List) 会包含它自己，让编译器无法确定大小。而 Box<List> 是指针（在 64 位系统上是 8 字节），大小是确定的。

2. Rc<T> — 引用计数

2.1 单所有权的问题

Rust 的每个值只有一个所有者。但有时一个值需要被多个地方共享——例如图数据结构中，一个节点可以有多个父节点。

use std::rc::Rc; let s = Rc::new(String::from("hello")); let s2 = Rc::clone(&s); // 引用计数 +1，不是深拷贝 let s3 = Rc::clone(&s); // 引用计数 +1，现在计数 = 3 println!("count = {}", Rc::strong_count(&s)); // 3

Rc<T> 的关键特性：

  • 只读共享——Rc<T> 只提供不可变借用
  • 引用计数——每次 Rc::clone 增加计数，离开作用域时减少
  • 非线程安全——只能在单线程中使用（没有 Send trait）
  • 低成本克隆——Rc::clone 只复制指针和增加计数，不深度复制数据
Rc::clone vs 深拷贝：Rc::clone(&s) 和 s.clone() 在语法上一样，但行为完全不同！Rc::clone 只增加引用计数（O(1) 操作），而 String::clone 会复制整个字符串（O(n) 操作）。为清晰起见，建议始终使用 Rc::clone(&s) 而不是 s.clone()。

3. RefCell<T> — 内部可变性

3.1 运行时借用检查

Rust 的借用规则（一个可变引用或多个不可变引用）是编译时强制执行的。但 RefCell<T> 将这些检查推迟到运行时——称为"内部可变性"模式。

use std::cell::RefCell; let cell = RefCell::new(5); *cell.borrow_mut() += 1; // 可变借用（运行时检查） let value = cell.borrow(); // 不可变借用 println!("{}", *value); // 6

RefCell 的借用方法：

方法返回类型说明
borrow()Ref<T>不可变借用，多个可共存
borrow_mut()RefMut<T>可变借用，唯一独占
运行时 panic：如果 borrow 时已有活跃的 borrow_mut（或反过来），程序会在运行时 panic。不会出现编译错误，但会有运行时错误。这与普通引用（编译时检查）的关键区别——RefCell 把内存安全保证从编译时移到了运行时。

3.2 何时使用 RefCell？

RefCell<T> 适用于以下场景：

  • 需要修改不可变引用背后的数据（内部可变性）
  • 满足借用规则但编译器无法理解（复杂的所有权模式）
  • 单线程环境——RefCell 不是 Sync

4. Rc<RefCell<T>> — 共享可变性

组合 Rc<T>（共享所有权）和 RefCell<T>（内部可变性）能得到"共享可变数据"——多个所有者可以修改同一份数据。

use std::rc::Rc; use std::cell::RefCell; let value = Rc::new(RefCell::new(10)); let v2 = Rc::clone(&value); // 共享所有权 *value.borrow_mut() += 5; // 通过 RefCell 修改 println!("{}", *value.borrow()); // 15

Rc<RefCell<T>> 是多线程中 Arc<Mutex<T>> 的单线程等价物。理解这个组合对理解后续的并发编程至关重要。

5. 智能指针总结

智能指针所有权可变性线程安全检查时机
Box<T>单一所有权可变或不可变是编译时
Rc<T>共享所有权不可变否编译时
RefCell<T>单一所有权内部可变否运行时
Rc<RefCell<T>>共享所有权内部可变否运行时
多线程版本：在多线程场景下，用 Arc<T> 代替 Rc<T>（Arc = Atomic Reference Counting），用 Arc<Mutex<T>> 代替 Rc<RefCell<T>>。这是下一章的内容。
