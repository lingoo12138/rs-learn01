第07章：泛型与 trait

学习目标

完成本章后，你将能够：

  • 理解泛型的概念，编写泛型函数和泛型结构体
  • 定义 trait 并为类型实现 trait
  • 使用 trait 默认方法
  • 使用 impl Trait 语法和 where 子句编写 trait bound
  • 返回实现了某 trait 的类型
  • 使用派生（derive）trait 自动实现常见行为
  • 编写泛型与 trait bound 结合的代码
小提示：如果你有 Java 或 C# 背景，Rust 的泛型类似 Java 的泛型（<T>）或 C# 的模板，但 Rust 的泛型在编译时执行单态化（monomorphization）——每种具体类型都会生成一份独立的代码，没有运行时代价！trait 则类似于 Java 的 interface 或 C++ 的抽象类，但更灵活，可以为已有类型实现已有的 trait。

1. 泛型（Generics）

1.1 泛型函数

泛型允许我们编写可以处理多种类型的函数，而不需要为每种类型重复编写代码。

fn identity<T>(x: T) -> T { x } let a = identity(5); // T 被推导为 i32 let b = identity("hello"); // T 被推导为 &str let c = identity(3.14); // T 被推导为 f64

<T> 是类型参数的声明，放在函数名之后、参数列表之前。它表示"这个函数接受一个任意类型 T 的参数，并返回同类型的值"。

注意：泛型类型参数通常使用大写的单个字母命名：T（Type）、E（Error）、K（Key）、V（Value）等。这是 Rust 社区的惯例，但不是强制要求。

1.2 多个类型参数

泛型可以有多个类型参数。

fn swap<T, U>(a: T, b: U) -> (U, T) { (b, a) } let result = swap(42, "hello"); // result 的类型是 (&str, i32)

1.3 泛型结构体

结构体也可以使用泛型参数，让字段可以接受不同的类型。

struct Point<T> { x: T, y: T, } let int_point = Point { x: 5, y: 10 }; // Point<i32> let float_point = Point { x: 1.0, y: 2.0 }; // Point<f64>

如果 x 和 y 可能是不同类型，可以使用多个类型参数：

struct Pair<T, U> { first: T, second: U, }

2. trait（特征）

2.1 定义 trait

Trait 定义了一组方法签名（有时也包含默认实现），描述了某种行为。任何实现了该 trait 的类型都必须提供这些方法的具体实现（除非有默认实现）。

trait Greet { fn greet(&self) -> String; }

2.2 为类型实现 trait

使用 impl Trait for Type 语法为某个类型实现 trait。

struct Dog { name: String, } impl Greet for Dog { fn greet(&self) -> String { format!("Woof! I'm {}", self.name) } } struct Cat; impl Greet for Cat { fn greet(&self) -> String { format!("Meow!") } } let dog = Dog { name: "Buddy".into() }; let cat = Cat; println!("{}", dog.greet()); // "Woof! I'm Buddy" println!("{}", cat.greet()); // "Meow!"

2.3 默认实现

Trait 方法可以有默认实现。实现该 trait 时可以选择覆盖它，也可以直接使用默认实现。

trait Hello { fn hello(&self) -> String { "Hello!".to_string() } } // 使用默认实现，不需要重新实现 hello 方法 struct Person; impl Hello for Person {} // 覆盖默认实现 struct Robot; impl Hello for Robot { fn hello(&self) -> String { "Beep boop!".to_string() } } assert_eq!(Person.hello(), "Hello!"); assert_eq!(Robot.hello(), "Beep boop!");

3. Trait Bound（特征约束）

3.1 impl Trait 语法

使用 impl Trait 作为参数类型，表示"这个参数可以是任何实现了该 trait 的类型"。

fn say_greet(item: impl Greet) -> String { item.greet() } // 任何实现了 Greet 的类型都可以传入 say_greet(Dog { name: "Rusty".into() }); say_greet(Cat);

3.2 完整 Trait Bound 语法

对于更复杂的情况（如多个参数需要同一 trait，或需要额外约束），可以使用完整的 trait bound 语法：

// 单个 bound fn greet_twice<T: Greet>(item: T) -> String { format!("{} and again {}", item.greet(), item.greet()) }

3.3 where 子句

当 trait bound 较多时，使用 where 子句使代码更清晰：

fn greet_twice<T>(item: T) -> String where T: Greet, { format!("{} and again {}", item.greet(), item.greet()) }

3.4 返回 impl Trait

函数可以返回一个实现了某 trait 的类型，而不暴露具体类型。

fn make_greeter() -> impl Greet { Dog { name: "Rusty".into() } } // 调用者只知道返回值实现了 Greet trait let g = make_greeter(); println!("{}", g.greet()); // "Woof! I'm Rusty"
注意：返回 impl Trait 的类型必须在函数体内唯一确定（不能有时返回 Dog，有时返回 Cat）。如果需要返回多种类型，需要使用后面章节会介绍的 Box<dyn Trait>。

4. 派生 Trait（Derive）

Rust 可以自动为类型实现一些标准 trait，只需要在类型定义前加上 #[derive(...)] 属性。

// 自动实现 Debug（格式化输出）和 PartialEq（相等比较） #[derive(Debug, PartialEq)] struct Point { x: i32, y: i32, } let p1 = Point { x: 1, y: 2 }; let p2 = Point { x: 1, y: 2 }; println!("{:?}", p1); // 输出: Point { x: 1, y: 2 } assert_eq!(p1, p2); // PartialEq 允许 == 比较

常用的派生 trait 包括：

Trait作用
Debug允许使用 {:?} 格式化输出
PartialEq允许使用 == 和 != 比较
EqPartialEq 的强化版（额外保证自反性）
Clone允许显式克隆（.clone()）
Copy允许按位复制（隐式复制，要求所有字段都实现了 Copy）
Hash允许计算哈希值
Default提供 default() 方法创建默认值

5. 综合：泛型 + Trait Bound

泛型和 trait bound 结合使用，可以编写非常灵活且类型安全的代码。

// 泛型函数，T 必须实现 PartialOrd（可比较大小） fn max_of<T: PartialOrd>(a: T, b: T) -> T { if a > b { a } else { b } } assert_eq!(max_of(3, 7), 7); // i32 assert_eq!(max_of("abc", "xyz"), "xyz"); // &str
理解核心思路：泛型让代码可以适应多种类型，trait bound 确保这些类型支持必要的操作。泛型解决了"什么类型都能用"的问题，trait bound 解决了"用什么方法"的问题。二者结合，就是 Rust 静态多态（static dispatch）的基础。
