第11章：并发与异步基础

学习目标

完成本章后，你将能够：

  • 使用 thread::spawn 创建系统线程
  • 使用 join 等待线程完成
  • 使用 move 闭包将数据移动到线程中
  • 使用 mpsc 通道在线程间传递消息
  • 使用 Arc<Mutex<T>> 实现共享状态并发
  • 理解 Send 和 Sync trait 的含义
  • 编写简单的 async / await 异步函数
小提示：Rust 的并发模型是"无畏并发"（fearless concurrency）——编译器在编译时就能捕获数据竞争和其他并发 bug。如果你在 Go 中使用 goroutine 和 channel，Rust 的 thread::spawn + mpsc 会给你相似的感觉，但所有权系统提供了更强的安全性保证。

1. 线程（Threads）

1.1 创建线程

使用 thread::spawn 创建一个新的操作系统线程：

use std::thread; let handle = thread::spawn(|| { "hello from thread".to_string() }); let result = handle.join().unwrap(); println!("{}", result); // "hello from thread"

关键点：

  • thread::spawn 接受一个闭包作为参数
  • 返回 JoinHandle<T>，可以用来等待线程完成
  • handle.join() 会阻塞当前线程直到子线程完成
  • join() 返回 Result<T>，如果线程 panic 则返回 Err

1.2 move 闭包

线程中的闭包需要获得变量的所有权。使用 move 关键字强制将捕获的变量移动到闭包中：

let x = 5; let handle = thread::spawn(move || { x * 2 // x 的所有权被移动到线程中 }); let result = handle.join().unwrap(); println!("{}", result); // 10 // println!("{}", x); // 编译错误！x 的所有权已被移动
为什么需要 move？没有 move 时，闭包只会借用变量。但 Rust 无法保证线程中的借用生命周期——主线程可能先释放了变量！move 将变量所有权转移到闭包中，完全避免了生命周期问题。

2. 消息传递 — mpsc

2.1 单生产者单消费者

Rust 的标准库提供了 mpsc（Multiple Producer, Single Consumer——多生产者，单消费者）通道：

use std::sync::mpsc; use std::thread; let (tx, rx) = mpsc::channel(); thread::spawn(move || { tx.send("hello".to_string()).unwrap(); }); let msg = rx.recv().unwrap(); println!("{}", msg); // "hello"

通道的传输端 (tx) 和生产端 (rx)：

  • tx.send(value) — 发送消息，如果接收端已丢弃则返回 Err
  • rx.recv() — 阻塞接收，直到收到消息或所有发送端关闭
  • rx.try_recv() — 非阻塞接收，没有消息时立即返回 Err
  • rx.recv_timeout(dur) — 带超时的接收

2.2 多生产者消息

通过 tx.clone() 可以创建多个发送端：

let (tx, rx) = mpsc::channel(); let tx2 = tx.clone(); thread::spawn(move || { tx.send("A".to_string()).unwrap(); }); thread::spawn(move || { tx2.send("B".to_string()).unwrap(); }); for _ in 0..2 { println!("Got: {}", rx.recv().unwrap()); }
Channel vs 共享内存：Go 有"通过通信来共享内存，而不是通过共享内存来通信"的理念。Rust 两种方式都支持，但 mpsc 通道通常更安全——消息的所有权被转移到通道中，避免了数据竞争。这也是 Rust 中"无畏并发"的体现之一。

3. Arc<Mutex<T>> — 共享状态

3.1 Mutex：互斥锁

Mutex<T>（互斥锁）保证同时只有一个线程能访问数据：

use std::sync::Mutex; let m = Mutex::new(0); { let mut num = m.lock().unwrap(); // 获取锁，阻塞直到锁可用 *num += 1; // 修改数据 } // 锁在此处自动释放 println!("{}", *m.lock().unwrap()); // 1

3.2 Arc：原子引用计数

Mutex 需要被多个线程共享。Rc 不是线程安全的，需要用 Arc（Atomic Reference Counting）：

use std::sync::{Arc, Mutex}; use std::thread; let counter = Arc::new(Mutex::new(0)); let mut handles = vec![]; for _ in 0..5 { let c = Arc::clone(&counter); handles.push(thread::spawn(move || { let mut num = c.lock().unwrap(); *num += 1; })); } for h in handles { h.join().unwrap(); } println!("Result: {}", *counter.lock().unwrap()); // 5

3.3 Send 和 Sync

Rust 通过两个特殊 trait 保证并发安全：

Trait含义示例
Send所有权可以在线程间转移Arc<T>, Box<T>, String
Sync可以被多个线程同时共享引用Arc<T>, Mutex<T>
不是 Send 不是 Sync：Rc<T> 既不是 Send 也不是 Sync——它的引用计数不是原子操作。RefCell<T> 也不是 Sync——它的借用检查不是线程安全的。这些 trait 是自动推导的：如果一个类型的所有字段都是 Send/Sync，它自身也是。如果你不小心在多线程中使用 Rc，编译器会直接报错——这就是"无畏并发"的保证。

4. async/await 基础

4.1 异步函数

异步函数用 async fn 声明，返回一个 Future。Future 是"将来某个时刻会产生一个值的计算"：

async fn hello_async() -> String { "hello async".to_string() } // 异步函数需要执行器 (executor) 来运行 let result = futures_executor::block_on(hello_async()); println!("{}", result); // "hello async"

4.2 async 块

也可以使用 async 块创建 Future：

let future = async { "hello from async block".to_string() }; let result = futures_executor::block_on(future); println!("{}", result);
async 和线程的区别：线程由操作系统调度，开销较大（每个线程有自己的栈）。异步任务由运行时（如 tokio、async-std）调度，在单个线程上可以运行成千上万个任务。异步适合 I/O 密集型任务（网络请求、文件读写），线程适合 CPU 密集型任务。
