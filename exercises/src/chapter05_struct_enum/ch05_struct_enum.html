第05章：结构体与枚举

学习目标

完成本章后，你将能够：

  • 定义并实例化结构体（struct），包括普通结构体和元组结构体
  • 使用 impl 为结构体实现方法，理解 &self 和 &mut self
  • 理解 Option<T> 和 Result<T, E> 枚举的用途
  • 使用 match 和 if let 进行模式匹配
  • 定义带数据的枚举变体（enum variants with data）
小提示：结构体和枚举是 Rust 中自定义数据类型的两种核心方式。如果你熟悉面向对象语言，Rust 的 struct 类似于类（但没有继承），而 enum 则比大多数语言中的枚举强大得多——每个变体可以携带不同类型和数量的数据。结合 match 的模式匹配，Rust 的枚举类型让你能够表达"这个值可能是 A、B 或 C，并且每种情况都携带不同数据"。

1. 结构体（Struct）

1.1 定义结构体

使用 struct 关键字定义结构体，为每个字段指定名称和类型：

struct User { username: String, email: String, sign_in_count: u64, active: bool, }

1.2 实例化结构体

为每个字段指定值来创建结构体实例。字段的顺序不需要与定义一致。

let user = User { email: String::from("alice@example.com"), username: String::from("alice"), active: true, sign_in_count: 1, };

1.3 可变实例

整个结构体实例必须是可变的（Rust 不允许只修改特定字段）：

let mut user = User { /* ... */ }; user.email = String::from("new@example.com");

1.4 字段初始化简写

如果字段名和变量名相同，可以简写：

let email = String::from("a@b.com"); let username = String::from("alice"); let user = User { email, username, active: true, sign_in_count: 1 };

1.5 结构体更新语法

使用 .. 从另一个实例复制未显式赋值的字段：

let user2 = User { email: String::from("b@c.com"), ..user };

2. impl 方法

使用 impl 块为结构体定义方法（method）。方法的第一个参数通常是 &self（不可变借用）、&mut self（可变借用）或 self（获取所有权）。

struct Rectangle { width: u32, height: u32, } impl Rectangle { // 不可变借用——只读访问 fn area(&self) -> u32 { self.width * self.height } // 可变借用——可以修改字段 fn set_width(&mut self, w: u32) { self.width = w; } } let mut rect = Rectangle { width: 10, height: 20 }; println!("面积: {}", rect.area()); // 200 rect.set_width(15); println!("新面积: {}", rect.area()); // 300
注意：&self 是 self: &Self 的语法糖，&mut self 是 self: &mut Self 的语法糖。在方法调用时，Rust 会自动处理引用和解引用，你不需要像 C 那样手动取地址。此外，impl 块中可以定义关联函数（associated functions），即第一个参数不是 self 的函数，通常用作构造函数，例如 String::from()。

3. 元组结构体（Tuple Structs）

元组结构体有字段名，只有字段类型和顺序：

struct Color(i32, i32, i32); struct Point(i32, i32, i32); let black = Color(0, 0, 0); let origin = Point(0, 0, 0); println!("R: {}, G: {}, B: {}", black.0, black.1, black.2);

元组结构体的作用类似于给元组取了一个有意义的名字。即使 Color 和 Point 的内部结构相同，它们也是不同的类型，不能混用。

4. 枚举（Enum）

4.1 Option<T> 枚举

Option<T> 是 Rust 标准库中最重要的枚举之一，用于表示"可能有值也可能没有值"的情况。它替代了其他语言中的 null / nil / None。

enum Option<T> { Some(T), None, } let some_number = Some(42); // Option<i32> let some_string = Some("hello"); // Option<&str> let absent: Option<i32> = None; // 必须标注类型
// 使用 match 处理 Option fn unwrap_or_zero(x: Option<i32>) -> i32 { match x { Some(v) => v, None => 0, } }

4.2 Result<T, E> 枚举

Result<T, E> 用于可能失败的操作，返回成功值 Ok(T) 或错误值 Err(E)。

enum Result<T, E> { Ok(T), Err(E), } fn check_positive(n: i32) -> Result<String, String> { if n > 0 { Ok("positive".to_string()) } else { Err("not positive".to_string()) } }

4.3 带数据的枚举变体

Rust 的枚举变体（variants）可以携带不同类型的数据，这是和其他语言最大的区别：

enum Message { Quit, // 没有数据 Move { x: i32, y: i32 }, // 匿名结构体 Write(String), // 元组形式 ChangeColor(i32, i32, i32), // 多个值 } impl Message { fn call(&self) { // 方法体 } } let m = Message::Write(String::from("hello")); m.call();

5. match 与 if let

5.1 match — 穷尽性模式匹配

match 必须穷尽（exhaustive）所有可能的模式：

let msg = Message::Move { x: 10, y: 20 }; match msg { Message::Quit => println!("退出"), Message::Move { x, y } => println!("移动到 ({}, {})", x, y), Message::Write(text) => println!("写入: {}", text), Message::ChangeColor(r, g, b) => println!("颜色: {} {} {}", r, g, b), }

5.2 if let — 简洁的单模式匹配

当只关心一种模式时，if let 比 match 更简洁：

let config = Some(3u8); // 使用 match（显得有些啰嗦） match config { Some(max) => println!("最大值为 {}", max), _ => (), // 必须处理所有情况 } // 使用 if let（简洁） if let Some(max) = config { println!("最大值为 {}", max); }
穷尽性检查：Rust 的 match 必须覆盖所有可能性。如果你漏掉了一个变体，编译器会报错。这是一个强大的特性——想象一下，如果你给 Message 增加了一个新变体，编译器会告诉你所有使用 match 的地方，确保你不会遗漏新的处理逻辑。这被称为"穷尽性检查"（exhaustiveness checking）。

6. 综合示例

struct Rectangle { width: u32, height: u32, } impl Rectangle { // 计算面积 fn area(&self) -> u32 { self.width * self.height } // 判断是否能容纳另一个矩形 fn can_hold(&self, other: &Rectangle) -> bool { self.width >= other.width && self.height >= other.height } } let rect1 = Rectangle { width: 5, height: 5 }; let rect2 = Rectangle { width: 3, height: 3 }; println!("面积: {}", rect1.area()); // 25 println!("能容纳吗? {}", rect1.can_hold(&rect2)); // true
